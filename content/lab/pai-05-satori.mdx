## The Technical Build

Satori runs locally on Gray's Mac in a Docker container. Under the hood:

- **FastAPI + LiteLLM** — real-time streaming via SSE
- **Keyword-based model routing** — Qwen 2.5 72B (default), Claude 3.5 Sonnet (research), Mixtral 8x22B (speed), DeepSeek v3 (coding). Bracket override: `[coding] do this`
- **Direct TELOS access** — `MY_TELOS.md` mounted read-only from disk. No semantic queries. No chunked guesses. The full file, whole, every session. Same as Bodhi.
- **Dual-layer memory** — Open Brain (pgvector) for session decisions and thoughts, MemPalace (ChromaDB) for architectural context
- **Session persistence** — every conversation stored in SQLite. System prompt stored per session so memory context never degrades between message 1 and message 50.

She reads the same TELOS I do. She knows the NIDs verbatim. She knows the cardinal directions (@strength, @heart, @mind, @might). She knows she is never allowed to paraphrase what is sacred — and if she doesn't have something in her loaded context, she says so instead of fabricating.

That last one took several rounds of iteration to get right. But she holds it now.

## Why This Is a Force Multiplier

Here is the honest business case.

**Token savings.** Bodhi (Claude Code) is powerful but sessions have limits. Satori runs on OpenRouter at a fraction of the cost. Routine tasks, context lookups, project status checks, developer work — she handles these without burning the Claude Code budget.

**Session continuity.** Claude Code has a context window. When that window approaches its limit, work pauses while we restart. Satori doesn't have that problem. She is always on, always remembering, always ready to continue.

**The handoff protocol.** When Bodhi hits token limits, Gray runs `/wrap`. The ritual updates HANDOFF.md and drops a task brief into Satori's inbox. Gray opens `localhost:19300`, tells Satori to pick up the inbox — and the machine keeps running. No restart delay. No re-briefing. The work continues.

**The client template.** Every client who comes to us for a PAI deployment gets a Satori clone. She is the pristine template — clean, tested, identity-complete. First client: Craig Simms. His Satori will know his TELOS, his brand voice, his projects. It will run on his VPS.
